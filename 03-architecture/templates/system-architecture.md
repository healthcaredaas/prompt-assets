## 三、服务架构

### 3.1 服务拆分

| 服务名称 | 职责 | 技术栈 | 端口 |
|----------|------|--------|------|
| gateway-service | API网关 | Spring Cloud Gateway | 8080 |
| user-service | 用户服务 | Spring Boot | 8081 |
| order-service | 订单服务 | Spring Boot | 8082 |
| product-service | 商品服务 | Spring Boot | 8083 |
| payment-service | 支付服务 | Spring Boot | 8084 |

### 3.2 服务依赖

```
gateway-service
    ├── user-service
    ├── order-service
    │   ├── product-service
    │   ├── user-service
    │   └── payment-service
    ├── product-service
    └── payment-service
```

### 3.3 服务通信

| 通信方式 | 场景 | 技术 |
|----------|------|------|
| 同步调用 | 实时查询 | OpenFeign |
| 异步消息 | 异步处理 | RocketMQ |
| 事件驱动 | 业务解耦 | RocketMQ |

## 六、安全架构

### 6.1 安全层次

| 层次 | 安全措施 | 技术 |
|------|----------|------|
| 网络安全 | 防火墙/WAF | 阿里云WAF |
| 应用安全 | 认证授权 | JWT/OAuth2 |
| 数据安全 | 加密传输 | HTTPS/TLS |
| 接口安全 | 签名验证 | RSA签名 |

                               │
                               ▼
                          获取用户权限
                               │
                               ▼
                          权限校验

## 八、容灾方案

### 8.1 高可用设计

| 层级 | 高可用措施 |
|------|------------|
| 接入层 | 多机房部署，DNS轮询 |
| 应用层 | 多副本部署，自动故障转移 |
| 数据层 | 主从复制，自动切换 |

### 8.2 容灾切换

| 场景 | 切换方案 | RTO | RPO |
|------|----------|-----|-----|
| 应用故障 | K8s自动重启 | <1min | 0 |
| 数据库故障 | 主从切换 | <5min | <1min |
| 机房故障 | 流量切换 | <30min | <5min |
