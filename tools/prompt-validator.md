# Prompt 验证器

> 版本: v1.0 | 最后更新: 2026-02-26

本文档提供 Prompt 质量验证的标准和方法，帮助确保 Prompt 资产的质量。

---

## 一、验证目的

Prompt 验证器用于：
1. 确保 Prompt 格式规范
2. 验证 Prompt 内容完整性
3. 检查 Prompt 可用性
4. 提供改进建议

---

## 二、验证维度

### 2.1 结构完整性

| 检查项 | 要求 | 权重 |
|--------|------|------|
| 元信息 | 包含版本、角色、输入要求、输出格式 | 20% |
| 角色设定 | 明确AI扮演的角色和专业背景 | 15% |
| 上下文 | 提供足够的背景信息 | 15% |
| 任务描述 | 清晰的任务说明和步骤 | 20% |
| 输出要求 | 明确的输出结构和质量标准 | 15% |
| 示例 | 提供输入输出示例 | 10% |
| 相关文档 | 列出相关文档链接 | 5% |

### 2.2 内容质量

| 检查项 | 要求 | 权重 |
|--------|------|------|
| 清晰性 | 语言清晰，无歧义 | 25% |
| 完整性 | 信息完整，无遗漏 | 25% |
| 准确性 | 技术内容准确 | 25% |
| 可操作性 | 可直接使用 | 25% |

### 2.3 格式规范

| 检查项 | 要求 | 权重 |
|--------|------|------|
| Markdown格式 | 正确使用Markdown语法 | 30% |
| 标题层级 | 标题层级合理（不超过4级） | 20% |
| 代码块 | 代码块正确标注语言 | 20% |
| 表格 | 表格格式正确 | 15% |
| 链接 | 链接有效可访问 | 15% |

## 四、验证评分

### 4.1 评分标准

| 分数范围 | 等级 | 说明 |
|----------|------|------|
| 90-100 | 优秀 | 完全符合要求，可直接使用 |
| 80-89 | 良好 | 基本符合要求，有小问题 |
| 70-79 | 合格 | 大部分符合要求，需改进 |
| 60-69 | 待改进 | 存在较多问题，需修改 |
| <60 | 不合格 | 存在严重问题，需重写 |

### 4.2 评分计算

```
总分 = 结构完整性得分 * 40% + 内容质量得分 * 40% + 格式规范得分 * 20%
```

## 六、常见问题

### 6.1 结构问题

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| 元信息缺失 | 缺少版本、角色等元信息 | 添加完整的元信息块 |
| 角色模糊 | 角色定义不清晰 | 明确角色名称和能力 |
| 任务不清 | 任务描述模糊 | 分步骤详细描述 |

### 6.2 内容问题

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| 上下文不足 | 背景信息不够 | 补充技术栈、业务背景 |
| 示例缺失 | 无输入输出示例 | 添加高质量示例 |
| 格式不对 | 输出格式不明确 | 提供输出模板 |

### 6.3 格式问题

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| 标题层级乱 | 标题层级不规范 | 统一标题层级 |
| 代码块无语言 | 代码块未标注语言 | 添加语言标识 |
| 表格格式错 | 表格对齐问题 | 使用正确的Markdown表格 |

## 八、验证工具

### 8.1 自动化检查脚本

```bash
#!/bin/bash
# prompt-validator.sh

PROMPT_FILE=$1

echo "===== Prompt 验证开始 ====="

# 检查文件是否存在
if [ ! -f "$PROMPT_FILE" ]; then
    echo "错误: 文件不存在"
    exit 1
fi

# 检查元信息
echo "1. 检查元信息..."
grep -q "## 元信息" "$PROMPT_FILE" && echo "  - 元信息块: 存在" || echo "  - 元信息块: 缺失"
grep -q "\*\*版本\*\*" "$PROMPT_FILE" && echo "  - 版本信息: 存在" || echo "  - 版本信息: 缺失"

# 检查角色设定
echo "2. 检查角色设定..."
grep -q "## 角色设定" "$PROMPT_FILE" && echo "  - 角色设定块: 存在" || echo "  - 角色设定块: 缺失"

# 检查上下文
echo "3. 检查上下文..."
grep -q "## 上下文" "$PROMPT_FILE" && echo "  - 上下文块: 存在" || echo "  - 上下文块: 缺失"

# 检查任务描述
echo "4. 检查任务描述..."
grep -q "## 任务描述" "$PROMPT_FILE" && echo "  - 任务描述块: 存在" || echo "  - 任务描述块: 缺失"

# 检查输出要求
echo "5. 检查输出要求..."
grep -q "## 输出要求" "$PROMPT_FILE" && echo "  - 输出要求块: 存在" || echo "  - 输出要求块: 缺失"

# 检查示例
echo "6. 检查示例..."
grep -q "## 示例" "$PROMPT_FILE" && echo "  - 示例块: 存在" || echo "  - 示例块: 缺失"

# 统计代码块
CODE_BLOCKS=$(grep -c '```' "$PROMPT_FILE" | awk '{print $1/2}')
echo "7. 代码块数量: $CODE_BLOCKS"

# 统计表格
TABLES=$(grep -c '^|' "$PROMPT_FILE")
echo "8. 表格行数: $TABLES"

echo "===== Prompt 验证完成 ====="
```

### 8.2 使用方法

```bash
# 验证单个Prompt文件
./prompt-validator.sh path/to/prompt.md

# 验证所有Prompt文件
find ./prompt-assets -name "*.md" -exec ./prompt-validator.sh {} \;
```

## 九、最佳实践

### 9.1 Prompt编写规范

1. **结构统一**: 所有Prompt使用统一的结构模板
2. **内容精炼**: 避免冗余，突出重点
3. **示例丰富**: 提供高质量的示例
4. **持续迭代**: 根据使用反馈持续改进

### 9.2 验证最佳实践

1. **定期验证**: 定期对Prompt进行验证
2. **版本管理**: 保留历史版本，记录改进过程
3. **用户反馈**: 收集用户反馈用于改进
4. **自动化**: 使用自动化工具提高效率
