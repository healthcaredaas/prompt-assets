# 测试阶段资产

本目录包含测试阶段的 Prompt 资产，帮助测试工程师进行测试计划和测试用例设计。

## 目录结构

```
05-testing/
├── README.md                  # 本文件
├── templates/                 # 模板文件
│   ├── test-plan.md           # 测试计划模板
│   └── test-case.md           # 测试用例模板
└── prompts/                   # Prompt文件
    ├── test-case-generation.md    # 测试用例生成
    └── integration-test.md        # 集成测试设计
```

## 使用场景

### 1. 测试计划编写
当开始新的测试项目时，使用 `templates/test-plan.md` 编写测试计划。

**输入**: 需求文档、设计文档
**输出**: 标准化的测试计划文档

### 2. 测试用例生成
根据需求生成测试用例，使用 `prompts/test-case-generation.md`。

**输入**: 功能需求、验收标准
**输出**: 完整的测试用例列表

### 3. 集成测试设计
设计系统集成测试方案，使用 `prompts/integration-test.md`。

**输入**: 系统架构、接口文档
**输出**: 集成测试方案

## 模板说明

### 测试计划模板 (templates/test-plan.md)
测试计划的标准模板，包含：
- 测试范围与目标
- 测试策略
- 测试环境
- 测试进度计划
- 风险识别

### 测试用例模板 (templates/test-case.md)
测试用例的标准格式，包含：
- 用例基本信息
- 前置条件
- 测试步骤
- 预期结果
- 测试数据

## 最佳实践

1. **测试左移**: 尽早开始测试设计，与开发并行
2. **覆盖率优先**: 确保核心功能测试覆盖率达到100%
3. **自动化优先**: 可自动化的测试用例优先设计
4. **数据驱动**: 使用参数化测试提高测试效率
