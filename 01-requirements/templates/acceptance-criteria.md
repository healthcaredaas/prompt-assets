# 验收标准模板

> 版本: v1.0 | 最后更新: 2026-02-26

本文档定义验收标准的编写规范和格式。

---

## 验收标准格式

### Given-When-Then 格式

验收标准采用行为驱动开发 (BDD) 的 Given-When-Then 格式：

```
Given [前置条件/初始状态]
When [触发动作/用户行为]
Then [预期结果/系统响应]
```

### 完整格式

```markdown
## AC-{{编号}}: {{验收标准标题}}

### 前置条件 (Given)
- 条件1: {{前置条件描述}}
- 条件2: {{前置条件描述}}
- 数据准备: {{测试数据描述}}

### 触发动作 (When)
- 用户操作: {{用户执行的操作}}
- 触发方式: {{手动触发/自动触发/接口调用}}

### 预期结果 (Then)
- 系统行为: {{系统的响应行为}}
- 数据变化: {{数据的预期变化}}
- 界面展示: {{界面的预期展示}}

### 异常场景
- 异常1: {{异常场景描述}} → {{预期处理}}
- 异常2: {{异常场景描述}} → {{预期处理}}

### 测试用例ID
- TC-{{编号}}: {{关联的测试用例}}
```

## 验收标准类型

### 1. 功能验收标准

验证功能是否按预期工作。

```markdown
## AC-001: 用户登录成功

### 前置条件 (Given)
- 用户已注册，账号状态正常
- 用户未登录

### 触发动作 (When)
- 用户输入正确的用户名: test@example.com
- 用户输入正确的密码: Password123
- 用户点击"登录"按钮

### 预期结果 (Then)
- 系统验证账号密码成功
- 生成JWT Token
- 跳转到系统首页
- 显示用户信息
```

### 2. 边界验收标准

验证边界条件的处理。

```markdown
## AC-002: 用户名长度边界

### 前置条件 (Given)
- 用户在注册页面

### 触发动作 (When)
- 输入用户名长度为 4 字符
- 输入用户名长度为 20 字符
- 输入用户名长度为 3 字符
- 输入用户名长度为 21 字符

### 预期结果 (Then)
- 长度 4 字符: 通过验证
- 长度 20 字符: 通过验证
- 长度 3 字符: 提示"用户名长度不能少于4个字符"
- 长度 21 字符: 提示"用户名长度不能超过20个字符"
```

### 3. 异常验收标准

验证异常场景的处理。

```markdown
## AC-003: 登录失败处理

### 场景1: 密码错误
### 前置条件 (Given)
- 用户已注册

### 触发动作 (When)
- 用户输入正确的用户名
- 用户输入错误的密码
- 用户点击"登录"按钮

### 预期结果 (Then)
- 提示"账号或密码错误"
- 登录失败次数 +1
- 不跳转页面

### 场景2: 账号锁定
### 前置条件 (Given)
- 用户已注册
- 用户已连续5次输入错误密码

### 触发动作 (When)
- 用户再次尝试登录

### 预期结果 (Then)
- 提示"账号已锁定，请30分钟后再试"
- 禁止登录
```

### 4. 性能验收标准

验证性能指标。

```markdown
## AC-004: 接口响应性能

### 前置条件 (Given)
- 系统正常运行
- 网络延迟 < 50ms

### 触发动作 (When)
- 并发用户数: 1000
- 持续时间: 5分钟
- 请求: 查询订单列表接口

### 预期结果 (Then)
- P50 响应时间 < 100ms
- P90 响应时间 < 200ms
- P99 响应时间 < 500ms
- 错误率 < 0.1%
- 系统资源: CPU < 70%, 内存 < 80%
```

### 5. 安全验收标准

验证安全要求。

```markdown
## AC-005: SQL注入防护

### 前置条件 (Given)
- 系统部署安全防护措施

### 触发动作 (When)
- 在用户名输入框输入: ' OR '1'='1
- 在密码输入框输入: ' OR '1'='1
- 点击登录按钮

### 预期结果 (Then)
- 系统返回"账号或密码错误"
- 不返回数据库错误信息
- 记录安全日志
- 不发生SQL注入
```

### 6. 兼容性验收标准

验证兼容性要求。

```markdown
## AC-006: 浏览器兼容性

### 前置条件 (Given)
- 应用部署完成

### 触发动作 (When)
- 在以下浏览器中访问应用:
  - Chrome 90+
  - Safari 14+
  - Firefox 88+
  - Edge 90+

### 预期结果 (Then)
- 所有浏览器界面渲染正常
- 所有功能正常运行
- 无JavaScript错误
- 响应式布局适配
```

## 验收标准编写规范

### 1. 原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **明确性** | 描述清晰无歧义 | ✅ 显示"登录成功"提示<br>❌ 显示成功提示 |
| **可验证性** | 可以被测试验证 | ✅ 跳转到 /dashboard<br>❌ 跳转到首页 |
| **完整性** | 覆盖所有场景 | 包括成功、失败、边界、异常 |
| **独立性** | 不依赖其他验收标准 | 每个AC独立可测 |

### 2. 避免的问题

| 问题 | 错误示例 | 正确示例 |
|------|----------|----------|
| 模糊描述 | 系统要快 | 接口响应时间 < 200ms |
| 隐含假设 | 用户登录 | Given 用户已注册且状态正常 |
| 不可测试 | 用户体验好 | 用户满意度评分 > 4.0 |
| 遗漏场景 | 只写成功场景 | 包含成功、失败、边界、异常 |

### 3. 数据准备

验收标准应包含明确的测试数据：

```markdown
### 测试数据

| 字段 | 测试值 | 说明 |
|------|--------|------|
| 用户名 | testuser01 | 正常用户 |
| 密码 | Pass@123 | 符合规则的密码 |
| 手机号 | 13800138000 | 有效手机号 |
| 邮箱 | test@example.com | 有效邮箱 |
```

## 验收标准模板库

### 登录相关

```markdown
## AC-LOGIN-001: 正常登录
- Given: 用户已注册，账号状态正常
- When: 输入正确账号密码并点击登录
- Then: 登录成功，跳转首页

## AC-LOGIN-002: 密码错误
- Given: 用户已注册
- When: 输入错误密码
- Then: 提示"账号或密码错误"，失败次数+1

## AC-LOGIN-003: 账号锁定
- Given: 连续5次密码错误
- When: 再次尝试登录
- Then: 提示"账号已锁定，请30分钟后再试"

## AC-LOGIN-004: 验证码登录
- Given: 用户已注册
- When: 输入手机号并获取验证码
- Then: 发送验证码，验证码5分钟有效

## AC-LOGIN-005: 登录过期
- Given: 用户已登录超过7天
- When: 访问需要登录的接口
- Then: 提示登录过期，跳转登录页
```

### 订单相关

```markdown
## AC-ORDER-001: 创建订单
- Given: 用户已登录，购物车有商品，库存充足
- When: 点击提交订单
- Then: 创建订单成功，库存预占，跳转支付页

## AC-ORDER-002: 库存不足
- Given: 商品库存为0
- When: 尝试下单
- Then: 提示"商品库存不足"，阻止下单

## AC-ORDER-003: 订单超时
- Given: 订单创建30分钟未支付
- When: 系统定时任务执行
- Then: 订单自动取消，库存释放

## AC-ORDER-004: 取消订单
- Given: 订单状态为待支付
- When: 用户点击取消订单
- Then: 订单状态变为已取消，库存释放

## AC-ORDER-005: 订单退款
- Given: 订单已支付
- When: 用户申请退款
- Then: 创建退款单，状态变为退款中
```

### 支付相关

```markdown
## AC-PAY-001: 支付成功
- Given: 订单待支付
- When: 支付成功回调
- Then: 订单状态变为已支付，库存扣减

## AC-PAY-002: 支付超时
- Given: 订单待支付超过30分钟
- When: 系统检查
- Then: 关闭支付，订单取消

## AC-PAY-003: 支付失败
- Given: 订单待支付
- When: 支付失败回调
- Then: 记录失败原因，订单保持待支付

## AC-PAY-004: 重复支付
- Given: 订单已支付
- When: 收到重复支付回调
- Then: 幂等处理，不重复处理
```

## 验收标准检查清单

编写验收标准时，使用以下检查清单确保质量：

### 功能性检查
- [ ] 覆盖正常流程
- [ ] 覆盖异常流程
- [ ] 覆盖边界条件
- [ ] 覆盖权限控制
- [ ] 覆盖数据校验

### 数据检查
- [ ] 明确测试数据
- [ ] 覆盖数据边界
- [ ] 覆盖数据格式
- [ ] 覆盖数据完整性

### 界面检查
- [ ] 明确界面展示
- [ ] 覆盖提示信息
- [ ] 覆盖错误展示
- [ ] 覆盖状态变化

### 性能检查
- [ ] 定义响应时间
- [ ] 定义并发能力
- [ ] 定义资源占用

### 安全检查
- [ ] 覆盖认证授权
- [ ] 覆盖数据安全
- [ ] 覆盖接口安全
